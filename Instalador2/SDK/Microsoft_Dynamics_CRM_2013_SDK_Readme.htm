Microsoft Dynamics CRM 2013 SP1 SDK Download Contents

The Microsoft Dynamics CRM 2013 SP1 SDK download package includes the following files and folders:

·         Bin Folder - Contains the assemblies needed for development by using Microsoft Dynamics CRM 2013 and Microsoft Dynamics CRM Online.

·         Resources Folder - Contains development resources, such as a style guide, images, data import maps, and other templates.

·         SampleCode Folder - Contains sample code to demonstrate how to develop by using Microsoft Dynamics CRM 2013 and Microsoft Dynamics CRM Online.

·         Schemas Folder - Contains the schemas used for validation by using Microsoft Dynamics CRM 2013 and Microsoft Dynamics CRM Online.

·         Solutions Folder - Contains sample solutions for use with Microsoft Dynamics CRM 2013 and Microsoft Dynamics CRM Online.

·         Tools Folder - Contains the tools needed to register plug-ins and workflows, and to create strongly typed classes.

·         Walkthroughs Folder - Contains the completed solutions that correspond to walkthroughs presented in the SDK documentation.

·         LicenseTerms.docx - The Microsoft software license terms for this SDK.

·         Other Redistributable.txt – Lists the redistributable files for Microsoft Dynamics CRM 2013.

·         Third Party Notices for CRM 2013 SDK.docx – Contains notices of third party code included in this SDK.

·         CrmSdk2013.chm - The help file that contains SDK documentation.

·         Microsoft_Dynamics_CRM_2013_SDK_Readme.htm - This readme file.

·         EntityMetadata.xlsx – Contains all the metadata information for a default installation.

·         Message-entity support for plug-ins.xlsx – Contains the listing of plug-in support per entity and message.

Feature Area Known Issues

CRM for Tablets

 Buttons are shown even when no records are selected (275319)

Issue: Buttons that use ribbon enable rules that have SelectionCountRule min=0 and max=0 are shown when a record is selected in Microsoft Dynamics CRM for tablets.

Workaround: Not using the SelectionCountRule will show the command when no records are selected. CRM for tablets provides contextual commands in the command bar based on the SelectionCountRule. 

Custom ribbon tabs are not always on the More menu (277703)

Issue: Custom tabs defined in the ribbon definition are not always on the More menu. CRM for tablets will place custom tabs into the command bar based on the sequence supplied in the definition. This is different from the web application, which always puts custom tabs on the More menu.

Workaround: To make the custom tabs show the same in both clients, change the sequence of the tab to be higher than the built-in tabs.

Command definitions that use MiscellaneousPrivilegeRule are not support (277848)

Issue: Command definitions that use MiscellaneousPrivilegeRule are not supported in this version of CRM for tablets.

Workaround: None.

CRM for tablets will hide commands in a ribbon DisplayRule that uses EntityPrivilege (279122)

Issue: When you use EntityPrivilege in a ribbon DisplayRule, CRM for tablets will hide commands that are not hidden in the web application. CRM for tablets evaluates both display and enable rules on the client side. When evaluations occur, both the data and user’s privileges are available. Therefore, DisplayRules that use entity privilege at the instance level will trim commands. In the web application, DisplayRules are not evaluated in a way where the record data is available. Therefore, DisplayRules do not trim based on instance-level privileges.

Workaround: None.

Calling setFocus in onLoad causes the read-only control to get focus (280307)

Calling setFocus in onLoad causes the read-only control to get focus as we keep the form in read-only state until the onLoad scripts are run.

Workaround: Clicking outside the control and clicking back on the control.

Customizing System defined Case Origin icon web resources is unsupported (305584)

While these web resources are hidden, their IsCustomizable property is set to True. Customizing them is not supported and can result in unexpected results.
